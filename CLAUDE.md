# 项目：智能体技能库

- 本项目备份我创建（或基于开源素材改编）的agent skills
- 除了本文件、软映射的 `AGENTS.md`以及 `/agent_tasks/`下的所有文件，其他所有文件，均推送到我的同名公开GitHub仓库

## 推送前检查

对每个需要推送的技能包，

- 如果在技能文件 `SKILL.md`的 `yaml`部分没有署名（如"author: WZM"），则自动加入
- 如果在技能文件以及附带的参考文档、脚本等内容中，有**任何涉及个人隐私**的内容，则暂时不推送，向我发出风险提示和修改建议
  - 通常我会手动修改，也可能对你施加指令
  - 当我确认可以推送，再次执行检查并向我报告检查结果，但直接推送
- 除上所述，不允许对 `/skills/`下的任何文件做任何改动

## 推送前执行

### 说明和索引

- `README_zh.md`基本上是我手动维护的中文说明文档，你需要将其内容翻译为英文，并写入 `README.md`，每次修订只需要检查更新内容即可
- `INDEX.md`是一个面向用户（可能是人类或者AI Agent）的索引文档，英文，每次新增或修改技能后检查、如有必要则更新
  - 每个技能写一节，标题H2，如"## paddle-pdf-re"；然后用bullet points写以下内容
  - 若一个skill子文件夹下仅有 `SKILL.md`，则只需要从技能文档中 `yaml`部分提取，"- description: [...]"和"- author: [...]"（或"- source: [...]"）
  - 若还有其他附加内容，则按照技能文档中的说明理解其作用，写"- structure: "，然后在代码块里画树状图，加注释简要说明
- 翻译中文版 `INDEX_zh.md`，并自动更新

### 版本号

- 每一轮变更，标记和更新版本号，如 `x.y.z`，各级之间不构成进位关系
- 在正式开始前有 `[v0.0.z]`，确认任务和初始状态，并**建立Git管理**；`z`从1开始，用户再次要求则取2, …
- 正式执行，版本号从 `0.1.0`开始，每次修订都要推送到GitHub仓库
  - 每轮次修订在当前基础上**自动更新**修订号 `z`，如从 `0.1.0`到 `0.1.1`, ..., `0.1.10`，`0.1.11`, ...
  - **当且仅当**我明确提示进入下一步（step），你才可以更新次版本号 `y`，如从 `0.1.10019`到 `0.2.0`
  - **当且仅当**我明确提示进入下一阶段（phase），才可以更新主版本号 `x`
  - 注意：各阶段之间并非完全线性，比如可能在[`v1.y.z`]到某一轮后，要继续[`v0.y.z`]，用户提示为准；若无提示，每次默认仅更新 `z`

### 工作日志

- 按照用户要求规划、修订与最终交付任务产出的同时，你**总是**需要记录**项目工作日志**，注明每次更新时间（`%Y-%m-%d %H:%M`）
- 在 `agent_tasks`下，
  - `ChangeLog.md`：在上次修订的基础上追写记录每次修订的新增、修复内容
  - `PathFile.md`：每次修订后更新文件结构树状图，简要记录**每一个**子文件夹及其下每一个文件的内容/功能以及修订内容
  - `Walkthrough.md`：注意！这是**最重要**的日志
    - 在开始执行前 [`v0.0.z`]，确认任务和初始可用材料（路径树状图），项目规划
    - 编排和检查任务列表，追写记录**每一轮**的任务进展和反思审查
    - 回顾反思所有任务是否完成，根据现有的产出和日志，总结执行过程，并说明交付产出的路径（树状图）
